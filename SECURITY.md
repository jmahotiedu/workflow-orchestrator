# Security Policy

## Supported Version

Current `main` branch is supported for this project.

## Reporting a Vulnerability

1. Submit privately through GitHub security advisories when enabled.
2. If unavailable, open an issue with prefix `[security]` and limited exploit detail.
3. Provide impacted service (`control-plane`, `worker`, `ui`), severity, and reproduction steps.

## Response Targets

- Acknowledgement: within 72 hours.
- Triage and fix plan: within 7 days.

## Scope Notes

- Default demo tokens are intentionally non-production.
- Treat `.env`, database dumps, and benchmark traces as sensitive.
